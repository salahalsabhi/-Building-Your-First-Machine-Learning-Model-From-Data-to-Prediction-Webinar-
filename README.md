# What is Machine Learning?

Machine Learning (ML) is a branch of Artificial Intelligence that enables computers to learn patterns directly from data instead of relying on manually programmed rules.

Instead of explicitly telling a computer every possible scenario, developers provide historical examples, allowing algorithms to identify relationships and make future predictions automatically.

The webinar used a simple analogy:

Imagine teaching a child to recognize cats.

Rather than explaining every possible characteristic (whiskers, ears, eyes, tail, fur), you simply show thousands of cat pictures.

Eventually, the child naturally learns to recognize cats independently.

Machine Learning works in exactly the same way.

Instead of programming every rule manually, we provide the computer with many examples so it can discover hidden patterns within the data. :contentReference[oaicite:2]{index=2}

# What Happens During Training?

During training the model repeatedly:

1. Reads the training data.
2. Makes an initial prediction.
3. Measures the error.
4. Adjusts internal parameters.
5. Repeats the process until performance improves.

This iterative learning process continues until the model reaches an acceptable level of accuracy.

---

# Why Testing Matters

A model may perform extremely well during training while failing on completely new data.

Testing helps answer the question:

> "Can the model generalize to unseen examples?"

This is why the dataset is divided into training and testing portions before learning begins.

# Model Evaluation

After training, the model must be evaluated objectively.

The webinar introduced several commonly used evaluation metrics. :contentReference[oaicite:2]{index=2}

# Accuracy

Accuracy measures the percentage of predictions that were correct.

Formula:

```
Correct Predictions
────────────────────
Total Predictions
```

Example:

```
100 Predictions

95 Correct

Accuracy = 95%
```

Advantages:

- Easy to understand
- Useful for balanced datasets

Limitations:

High accuracy can still be misleading when classes are heavily imbalanced.

---

# Precision

Precision answers the question:

> "Out of everything predicted as positive, how many actually were positive?"

Example:

```
Predicted Fraud

↓

Actually Fraud
```

High precision reduces false alarms.

This is especially important in:

- Fraud Detection
- Malware Detection
- Medical Diagnosis

---

# Recall

Recall answers a different question:

> "Out of all actual positive cases, how many did the model successfully find?"

Example:

```
Actual Malware

↓

Detected Malware
```

High recall means fewer dangerous cases are missed.

Important applications:

- Intrusion Detection
- Cancer Detection
- Threat Hunting

---

# F1 Score

Precision and Recall often compete.

Increasing one may reduce the other.

The F1 Score combines both into a single balanced metric.

Higher F1 Scores generally indicate better overall classification performance.

---

# Confusion Matrix

The webinar also introduced the Confusion Matrix as a way of understanding prediction results.

Example:

|                | Predicted Positive | Predicted Negative |
|----------------|------------------:|------------------:|
| Actual Positive | True Positive | False Negative |
| Actual Negative | False Positive | True Negative |

The confusion matrix allows practitioners to understand exactly where prediction errors occur instead of relying solely on accuracy.

# 4. Supervised Machine Learning

Supervised Learning is the most common category of Machine Learning.

In supervised learning, the model is trained using **labeled data**, meaning every training example already contains the correct answer.

The objective is to learn the relationship between:

```
Input (Features) → Output (Label)
```

Once trained, the model can make predictions on completely unseen data.

---

## Example

Suppose we want to predict house prices.

Training Data:

| Size (m²) | Bedrooms | Price |
|-----------|-----------|---------|
| 90 | 2 | £180,000 |
| 120 | 3 | £250,000 |
| 150 | 4 | £320,000 |

The model learns patterns such as:

- Larger houses generally cost more.
- More bedrooms usually increase value.
- Location may also influence price.

When a new house appears:

```
135 m²
3 Bedrooms
```

The model predicts:

```
≈ £280,000
```

### Decision Trees

Decision Trees make decisions by asking a sequence of questions.

Example:

```

       Yes
      /
 Owns House?

  Yes     No
```

Disadvantages:

### Random Forest

Random Forest combines many Decision Trees.

Instead of relying on one tree:

```
Tree 1

Tree 2

Tree 3

Tree 4

Tree 5

↓

Majority Vote

↓

Prediction
```

Advantages:

- Higher accuracy
- More robust
- Less overfitting

Widely used in production systems.

---

### Support Vector Machine (SVM)

SVM attempts to separate different classes by finding the optimal boundary.

Example:

```
Cats ● ● ●

------------------

Dogs ▲ ▲ ▲
```

The line separating both classes is called the **Hyperplane**.

Used for:

- Image recognition
- Face recognition
- Text classification

---

### k-Nearest Neighbors (k-NN)

k-NN classifies a new data point based on its nearest neighbors.

Example:

```
New Customer

↓

Look at 5 nearest customers

↓

Most common class wins
```

Advantages:

- Very simple
- No training phase

Disadvantages:

- Slow with large datasets
- Sensitive to feature scaling

# Part 5 — Data Preprocessing

Machine Learning models rely heavily on high-quality data. Before training a model, raw data must undergo preprocessing to improve consistency, remove errors, and prepare it for learning.

---

## Why Data Preprocessing Matters

Real-world datasets often contain:

- Missing values
- Duplicate records
- Incorrect data types
- Outliers
- Inconsistent formatting
- Irrelevant features

Poor-quality data results in poor model performance.

This follows the principle:

> Garbage In → Garbage Out (GIGO)

---

## Common Data Preprocessing Steps

### Handling Missing Values

Examples:

- Replace missing numbers with the mean or median.
- Replace missing categorical values with the most frequent category.
- Remove rows containing excessive missing data.

---

### Removing Duplicate Records

Duplicate observations can bias the learning process.

Example:

```python
df.drop_duplicates()
```

---

### Encoding Categorical Variables

Machine Learning algorithms require numerical input.

Example:

| Colour | Encoding |
|---------|----------|
| Red | 0 |
| Blue | 1 |
| Green | 2 |

Techniques include:

- Label Encoding
- One-Hot Encoding

---

### Feature Scaling

Many algorithms perform better when features are on similar scales.

Common methods:

### Standardization

Transforms values around mean = 0 and standard deviation = 1.

### Normalization

Scales values between 0 and 1.

---

### Feature Selection

Not every column improves predictions.

Selecting only relevant features can:

- Reduce overfitting
- Improve accuracy
- Increase training speed

# Part 9 — Evaluating Model Performance

Average prediction error.

### Mean Squared Error (MSE)

Squares prediction errors.

Penalizes large mistakes.

### Root Mean Squared Error (RMSE)

### R² Score

## Classification Metrics

Overall correct predictions.

### F1 Score

Excellent training accuracy.

Poor performance on new data.

- Irrelevant features

# Evaluating Model Performance

Building a model is only the beginning.

A model must also be evaluated to determine how well it performs.

---

## Regression Metrics

### Mean Absolute Error (MAE)

Measures the average absolute prediction error.

Lower values indicate better performance.

---

### Mean Squared Error (MSE)

Squares each prediction error.

Large mistakes receive heavier penalties.

---

### Root Mean Squared Error (RMSE)

Provides the prediction error using the same units as the original data.

---

### R² Score

Measures how much of the variance is explained by the model.

Values range between:

```
0 → Poor model

1 → Perfect model
```

---

## Classification Metrics

For classification problems, different evaluation methods are used.

---

### Accuracy

Percentage of correct predictions.

```
Correct Predictions

--------------------

Total Predictions
```

---

### Precision

Measures how many predicted positives were actually correct.

Important for:

- Spam detection
- Fraud detection

---

### Recall

Measures how many real positive cases were successfully detected.

Important in:

- Medical diagnosis
- Intrusion Detection Systems

---

### F1 Score

Balances Precision and Recall.

Useful when datasets are imbalanced.

---

### Confusion Matrix

```text
                    Predicted

               Positive   Negative

Actual Positive     TP        FN

Actual Negative     FP        TN
```

Definitions:

- TP = True Positive
- TN = True Negative
- FP = False Positive
- FN = False Negative

---

# Model Generalization

A good Machine Learning model should perform well on data it has never seen before.

This ability is called **Generalization**.

---

## Underfitting

Characteristics:

- Model too simple
- Poor accuracy
- Cannot learn patterns

Example:

```text
Data

● ● ● ● ●

Simple Line

──────────
```

The model misses important relationships.

---

## Overfitting

Characteristics:

- Memorizes training data
- Excellent training accuracy
- Poor testing accuracy

Example:

```text
Data

●●●●●●●

Curve

~~~~~^^^^^^~~~~
```

The model becomes overly complex.

---

## Good Fit

The ideal model balances:

- Learning
- Simplicity
- Generalization

# Hyperparameter Tuning

Hyperparameters control how the model learns.

Examples include:

- Learning rate
- Tree depth
- Number of neighbors
- Number of estimators
- Batch size

Unlike model parameters, hyperparameters are chosen before training begins.

---

## Grid Search

Grid Search systematically tests multiple hyperparameter combinations.

Example:

```text
Tree Depth

3

5

7

9
```

The combination producing the highest validation accuracy is selected.

---

## Cross Validation

Cross Validation evaluates the model across multiple dataset splits.

Typical approach:

```
5-Fold Cross Validation

Dataset

↓

Split into five sections

↓

Train on four

↓

Test on one

↓

Repeat five times

↓

Average Score
```

Advantages:

- More reliable evaluation
- Reduces bias
- Better use of limited datasets
